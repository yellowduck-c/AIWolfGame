# 项目记忆机制设计分析

## 1. 设计目标

该项目中的“记忆机制”不是通用聊天机器人那种长期人格记忆，而是**服务于狼人杀博弈的信息隔离记忆系统**。它的核心目标有三个：

1. **保存角色在局内可见的信息**，支持后续发言、投票、技能决策。
2. **严格区分信息层级**，避免不同阵营或不同身份读取不该知道的信息。
3. **将历史观察与自身行动一起打包进决策输入**，供 LLM 在每一步做条件化推理。

从实现上看，记忆机制主要由以下几部分组成：

- `werewolf_backend/agent/state/memory.py`
- `werewolf_backend/agent/state/schemas.py`
- `werewolf_backend/game_engine/decision_context.py`
- `werewolf_backend/agent/roles/base_agent.py`

## 2. 核心数据结构

### 2.1 AgentIdentityMemory

位置：[memory.py:7-12](werewolf_backend/agent/state/memory.py#L7-L12)

```python
@dataclass(slots=True)
class AgentIdentityMemory:
    player_id: int
    role: str
    camp: str
```

这是最基础的身份记忆，保存：

- 玩家编号 `player_id`
- 角色 `role`
- 阵营 `camp`

它不参与复杂推理，但作为记忆摘要的一部分，会持续告诉模型“我是谁”。

### 2.2 AgentDecisionHistory

位置：[memory.py:14-18](werewolf_backend/agent/state/memory.py#L14-L18)

```python
@dataclass(slots=True)
class AgentDecisionHistory:
    speeches: list[str] = field(default_factory=list)
    votes: list[int] = field(default_factory=list)
    skills: list[dict[str, Any]] = field(default_factory=list)
```

该结构记录 agent 自己已经做出的历史动作：

- 发过什么言
- 投过谁
- 用过什么技能

这是“**自我行为历史**”层，不是外部世界状态。

### 2.3 AgentMemory

位置：[memory.py:21-57](werewolf_backend/agent/state/memory.py#L21-L57)

```python
@dataclass(slots=True)
class AgentMemory:
    identity: AgentIdentityMemory
    public_observations: list[dict[str, Any]] = field(default_factory=list)
    private_facts: list[dict[str, Any]] = field(default_factory=list)
    decision_history: AgentDecisionHistory = field(default_factory=AgentDecisionHistory)
```

这是整个记忆系统的核心容器，分成三层：

1. `public_observations`
   - 所有人都能看到的公共事件
   - 例如：谁发言、谁出局、阶段切换等

2. `private_facts`
   - 仅当前 agent 可知的私有事实
   - 例如：
     - 预言家查验结果
     - 女巫药剂是否已用
     - 狼人私聊记录
     - 猎人是否已开枪

3. `decision_history`
   - 当前 agent 自己的动作历史

该结构还提供四类写入函数：

- `record_public_observation()`
- `record_private_fact()`
- `record_speech()`
- `record_vote()`
- `record_skill()`

以及一个统一导出接口：

```python
def to_summary(self) -> dict[str, Any]:
```

它会把身份、公共观察、私有事实、行为历史整体序列化，作为 `memory_summary` 注入决策输入。

## 3. 记忆如何进入决策输入

### 3.1 AgentDecisionInput 是统一决策载体

位置：[schemas.py:7-19](werewolf_backend/agent/state/schemas.py#L7-L19)

```python
@dataclass(slots=True)
class AgentDecisionInput:
    agent_id: int
    role: str
    camp: str
    phase: str
    round: int
    public_state: dict[str, Any]
    private_state: dict[str, Any]
    camp_shared_state: dict[str, Any]
    memory_summary: dict[str, Any]
    legal_actions: dict[str, Any]
    specialization: dict[str, Any] = field(default_factory=dict)
```

这个结构很关键，它把一次决策的上下文拆为五大块：

- `public_state`：公共状态
- `private_state`：该角色私有状态
- `camp_shared_state`：阵营共享状态
- `memory_summary`：历史记忆摘要
- `legal_actions`：当前允许动作

可以理解为：

- `public/private/camp_shared` 主要描述“当前世界”
- `memory_summary` 主要描述“过去我经历了什么”

### 3.2 build_agent_decision_input 的职责

位置：[decision_context.py:148-165](werewolf_backend/game_engine/decision_context.py#L148-L165)

该函数负责在每次需要发言/投票/用技能时，临时构建一份新的 `AgentDecisionInput`。

```python
return AgentDecisionInput(
    agent_id=agent_snapshot["id"],
    role=agent_snapshot["role"],
    camp=agent_snapshot["camp"],
    phase=session["phase"],
    round=session["round"],
    public_state=_build_public_state(session),
    private_state=_build_private_state(session, agent_snapshot, agent),
    camp_shared_state=_build_camp_shared_state(session, agent_snapshot, agent),
    memory_summary=agent.memory.to_summary(),
    legal_actions=legal_actions,
)
```

这意味着记忆不是零散读取，而是**在决策边界统一打包**。

## 4. 三层信息隔离设计

这是本项目记忆机制最重要的地方。

### 4.1 公共信息：public_state

位置：[decision_context.py:30-42](werewolf_backend/game_engine/decision_context.py#L30-L42)

包含：

- 当前 phase
- 当前 round
- 当前存活玩家列表
- `public_events`

这是所有角色都共享的世界模型。

### 4.2 私有信息：private_state

位置：[decision_context.py:45-65](werewolf_backend/game_engine/decision_context.py#L45-L65)

包含：

- `self`：自己的 id / role / camp / status
- `private_facts`：当前 agent 私有事实
- 某些角色特有的技能上下文

例如：

- 女巫会拿到 `witch_resources`
- 女巫还会拿到 `night_context.wolf_target_id`
- 有技能的角色会拿到 `skill`

这部分体现的是“**同一游戏状态在不同角色眼中是不同的**”。

### 4.3 阵营共享信息：camp_shared_state

位置：[decision_context.py:68-90](werewolf_backend/game_engine/decision_context.py#L68-L90)

只有狼人阵营会拿到非空值：

```python
if agent_snapshot["camp"] != "狼人":
    return {}
```

狼人可见内容包括：

- `teammates`
- `camp_chat_history`

其中 `camp_chat_history` 来源于该狼人私有记忆里、类型为 `camp_chat_observed` 的事实。

这套设计很漂亮的一点是：

- 狼人私聊不是放进 `public_events`
- 也不是做全局共享表直接给所有人看
- 而是通过“每只狼都观察到一条私有事实”再汇总成 camp-shared 视图

因此它本质上仍然遵守了“信息隔离”，只是对狼人阵营做了受控共享。

## 5. 记忆的写入路径

### 5.1 公共事件写入

位置：[base_agent.py:108-113](werewolf_backend/agent/roles/base_agent.py#L108-L113)

```python
def observe_public_event(self, event: dict[str, Any]) -> None:
    self.memory.record_public_observation(event)

def observe_private_fact(self, fact: dict[str, Any]) -> None:
    self.memory.record_private_fact(fact)
```

游戏引擎在事件发生后，会调用 agent 的观察接口，把状态写入记忆。

### 5.2 自身决策写入

位置：
- [base_agent.py:114-126](werewolf_backend/agent/roles/base_agent.py#L114-L126)
- [base_agent.py:128-140](werewolf_backend/agent/roles/base_agent.py#L128-L140)

例如：

- `speak()` 后记录 speech
- `vote()` 后记录 vote
- `use_skill()` 后记录 skill

这意味着 agent 后续不仅知道“外界发生了什么”，也知道“自己曾经怎么做过”。

### 5.3 私有角色事实写入

当前实现里，关键角色事实是在夜间逻辑里写入：

- 预言家查验结果：`seer_inspection`
- 女巫解药：`witch_heal_used`
- 女巫毒药：`witch_poison_used`
- 猎人开枪：`hunter_shot_used`
- 狼人私聊：`camp_chat_observed`

这些事实都进入 `private_facts`，后续再进入：

- `private_state`
- `memory_summary`
- 对狼人来说，还可进入 `camp_shared_state.camp_chat_history`

## 6. 狼人私聊是记忆机制的一个代表性案例

这是当前项目中最能体现“记忆 + 信息隔离 + 决策复用”的例子。

### 6.1 私聊生成阶段

夜晚开始时，狼人先进行 camp chat。

每条私聊：

- 对观战端可见
- 但不进入 `public_events`
- 同时写入所有存活狼人各自的 `private_facts`

写入形式大致是：

```python
{
  "type": "camp_chat_observed",
  "round": session["round"],
  "from_id": agent_snapshot["id"],
  "content": content,
}
```

### 6.2 后续刀人阶段复用

在狼人稍后进入 `use_skill()` 时，`build_agent_decision_input()` 会重新构造上下文，其中：

- `camp_shared_state.camp_chat_history`
  会从该狼的 `private_facts` 中筛出所有 `camp_chat_observed`

这样模型在刀人 prompt 里就能看到同伴刚才的私聊内容。

### 6.3 设计意义

这个机制说明项目的“记忆”不是静态档案，而是：

- **先观察**
- **再入记忆**
- **在下一步决策时重新编排进上下文**

这是一个明显的“事件驱动记忆注入”结构。

## 7. 当前记忆机制的特点总结

### 7.1 优点

1. **结构清晰**
   - 身份、公共观察、私有事实、自身行为历史分层明确。

2. **天然支持信息隔离**
   - 通过 `public_state / private_state / camp_shared_state` 三分模型，避免全量上下文泄漏。

3. **决策边界清楚**
   - 每次发言/投票/技能调用时，统一构建 `AgentDecisionInput`。

4. **容易扩展**
   - 新角色、新私有事实、新阵营共享信息都可以继续往这套结构里接。

5. **适合 LLM**
   - 记忆最终被序列化成 JSON 风格对象，便于 prompt 注入和规则化约束。

### 7.2 当前局限

1. **没有摘要压缩**
   - `public_observations`、`private_facts`、`decision_history` 都是线性累积。
   - 游戏回合增加后，prompt 可能越来越长。

2. **没有记忆优先级机制**
   - 关键事实和普通事件目前没有显式权重区分。

3. **没有显式遗忘机制**
   - 所有事实持续保留，长期可能造成噪声。

4. **阵营共享仍是派生视图，不是独立层**
   - 当前 `camp_shared_state` 是由私有事实现算出来的，而不是单独存储的 camp memory。

## 8. 可以如何继续演进

如果后续要增强这套机制，可以考虑：

### 8.1 引入短期/长期记忆分层

例如把：

- 本轮关键事件
- 历史较远的累计模式

分开存储，避免所有信息都平铺进 prompt。

### 8.2 引入记忆摘要器

在进入 prompt 前，对：

- 历史 public events
- 大量 speech history
- camp_chat_history

做规则摘要或模型摘要。

### 8.3 给 private_facts 加 schema

现在 `private_facts` 是开放 dict，灵活但不够强约束。
后续可以把常见 fact 类型结构化，提升可维护性与验证能力。

### 8.4 引入阵营共识提取层

比如专门从 `camp_chat_history` 中抽取：

- 当前统一目标
- 是否已形成共识
- 共识置信度

再喂给 skill prompt，这样会比让模型自己从聊天文本里再判断一次更稳定。

---

## 9. 结论

本项目的记忆机制本质上是一个**面向局内决策的分层上下文系统**，而不是通用对话 memory。

它的核心设计可以概括为：

- 用 `AgentMemory` 保存“我是谁、我看到了什么、我自己做过什么”；
- 用 `decision_context` 把这些记忆按权限拆成 `public / private / camp_shared` 三层；
- 在每次行动前统一组装成 `AgentDecisionInput`，送入提示词链路。

这套设计已经很好地服务了狼人杀这种强信息隔离、多角色异步决策的任务场景。其最突出的特点不是“记住更多”，而是“**只让每个 agent 记住它应该记住的东西，并在正确的行动节点拿到正确版本的上下文**”。
